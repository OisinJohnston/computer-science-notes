Functions

What is a function?

A function is a self contained piece of code that carries out a particular task.

Why use functions?

  • Allow us to do modular programming i.e. to seperate the program into independent modules or pieces - each of which performs a specific task

  • Reusable to assist in avoiding:

    • Wasting time retyping code
    • Copying ans pasting errors
    • Time updating the same piece of code in multiple places.

Function definition

Before we can use a function, we must define it

  • an easy way to define a function in javascript is using the function keyword followed by:

  • a unique function name followed by round brackets ()

  • A list of parameters if required

  • A statement block surrounded by curly braces

  • A return statement if required

The simplest way to define a function is this:

function function_name(parameter_1, parameter_2) {
    // the code in our function goes here
    console.log(parameter_1);
    return parameter_2; // we can also return values to the caller
}

Now if we want to call this function we simply do the following:

x = function_name("bob", "dylan")

console.log(x) // dylan

the arguments for a function are the values passed into the function

we can use the parameters passed into our function to influence the output:

function greeter(name) { 
    return "Hello, " + name
}

Here is a simple example of a function that squares a number:

function square(number) {
    return number**2
}

Pass by value and pass by reference

  • When you call a function and you type in the arguments, the arguments are said to be passed to the function where they are processed

  • There are two ways of doing this pass by value pass by reference

Pass by value

  • in Pass by value, a copy of the argument is made to a new memory location.

  • The values in this location are passed into the function and the original value is not changed by any processing done in the function.

pass by value example:

code:

function call_me(params) {
    params++
    console.log("params number is: ", params)
}

x = 10;
console.log("Starting number is: ", x);
call_me(x)
console.log("Ending number is: ", x);

output:

- Starting number is: 10

- params number is: 11

- Ending number is: 10

as you can see only the value of xis passed into the function and the global variable x never has its value changed this is because it is a number and number and all primitives in javascript operate on a pass by value basis.

Pass by reference

  • In pass by reference, the location of the argument in memory is passed into the function and the original value is changed by any processing done in the function.

  • There is no new memory used.

Arrays and other non-primitive types operate on a pass by reference basis:

example:

code:

function call_me(params) {
    for (i=0; i<params.length; i++) {
        params[i] = params[i]+1;
    }
    console.log("params ", typeof(params), "is: ", params)
}
x = [1,2,3,4,5]

console.log("Starting array is: ", x)
call_me(x)
console.log("Ending array is: ", x)

output:

Starting array is: [1, 2, 3, 4, 5]
params object is: [2, 3, 4, 5, 6]
Ending array is: [2,3, 4, 5, 6]
